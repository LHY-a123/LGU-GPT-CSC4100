# CUSZ-GPT 运行指南

## 快速开始（如果已配置好环境）

### 方式一：使用命令行查询（推荐）

#### 1. 确保 vLLM 服务正在运行

```bash
# 检查 vLLM 服务状态
curl http://localhost:8000/v1/models

# 如果服务未运行，启动 vLLM（多GPU模式）
bash scripts/start_vllm_multi_gpu.sh

# 或者单GPU模式
nohup python -m vllm.entrypoints.openai.api_server \
  --model /kongweiwen/models/Qwen2.5-14B-Instruct \
  --tensor-parallel-size 1 \
  --gpu-memory-utilization 0.9 \
  --port 8000 > vllm.log 2>&1 &
```

#### 2. 确保索引已构建

```bash
# 检查索引是否存在
ls -lh data/vector_db/lightrag/

# 如果索引不存在，构建索引
python3 -m rag_system.main build-index
```

#### 3. 运行查询

```bash
# 单次查询
python3 -m rag_system.main query \
  --query "香港中文大学（深圳）有哪些学院？" \
  --show-context

# 交互式查询（推荐）
python3 -m rag_system.main query --show-context
```

### 方式二：使用 Web UI（LightRAG Server）

#### 1. 启动 LightRAG Server

```bash
# 使用启动脚本（推荐）
bash scripts/start_lightrag_server.sh

# 或直接运行 Python 脚本
python3 scripts/start_lightrag_server.py
```

#### 2. 访问 Web UI

- 本地访问：http://localhost:8080
- 内网访问：http://<服务器IP>:8080
- API 文档：http://localhost:8080/docs

## 完整运行流程

### 步骤 1：启动 vLLM 服务

```bash
# 方式1：使用多GPU脚本（推荐，如果有多块GPU）
bash scripts/start_vllm_multi_gpu.sh

# 方式2：单GPU启动
nohup python -m vllm.entrypoints.openai.api_server \
  --model /kongweiwen/models/Qwen2.5-14B-Instruct \
  --tensor-parallel-size 1 \
  --gpu-memory-utilization 0.9 \
  --port 8000 \
  --host 0.0.0.0 \
  --trust-remote-code > vllm.log 2>&1 &

# 等待服务启动（约1-2分钟），然后检查
curl http://localhost:8000/v1/models
tail -f vllm.log  # 查看日志
```

### 步骤 2：构建向量索引（首次运行需要）

```bash
# 构建索引（需要 vLLM 服务运行）
python3 -m rag_system.main build-index

# 如果数据已更新，强制重新构建
python3 -m rag_system.main build-index --force-recreate

# 检查索引是否构建成功
ls -lh data/vector_db/lightrag/
```

### 步骤 3：运行查询

#### 方式 A：命令行查询

```bash
# 单次查询
python3 -m rag_system.main query \
  --query "你的问题" \
  --show-context

# 交互式查询（推荐）
python3 -m rag_system.main query --show-context

# 指定查询模式
python3 -m rag_system.main query \
  --query "你的问题" \
  --mode hybrid  # 可选：local, global, hybrid, naive, mix, bypass
```

#### 方式 B：Web UI

```bash
# 启动 Web UI
bash scripts/start_lightrag_server.sh

# 然后在浏览器访问 http://localhost:8080
```

## 常用命令

### 检查服务状态

```bash
# 检查 vLLM 服务
curl http://localhost:8000/v1/models

# 检查 GPU 使用情况
nvidia-smi

# 查看 vLLM 日志
tail -f vllm.log
```

### 停止服务

```bash
# 停止 vLLM 服务
pkill -f "vllm.entrypoints.openai.api_server"

# 或查找进程并杀死
ps aux | grep vllm
kill <PID>
```

### 重新构建索引

```bash
# 强制重新构建索引
python3 -m rag_system.main build-index --force-recreate
```

## 故障排查

### 问题1：vLLM 服务无法启动

```bash
# 检查 GPU
nvidia-smi

# 检查 CUDA
nvcc --version

# 检查模型路径
ls -lh /kongweiwen/models/Qwen2.5-14B-Instruct/

# 降低显存使用率
python -m vllm.entrypoints.openai.api_server \
  --model /kongweiwen/models/Qwen2.5-14B-Instruct \
  --gpu-memory-utilization 0.7 \
  --port 8000
```

### 问题2：索引构建失败

```bash
# 检查 vLLM 服务是否运行
curl http://localhost:8000/v1/models

# 检查数据文件是否存在
ls -lh data/rag/*.jsonl

# 查看详细错误信息
python3 -m rag_system.main build-index --vllm-base-url http://localhost:8000/v1
```

### 问题3：查询无响应

```bash

ls -lh data/vector_db/lightrag/

# 检查 vLLM 服务
curl http://localhost:8000/v1/models

# 尝试重新构建索引
python3 -m rag_system.main build-index --force-recreate
```

## 环境变量配置

确保以下环境变量已设置（在 `.env` 文件或 `~/.bashrc` 中）：

```bash
# vLLM 配置
export VLLM_BASE_URL=http://localhost:8000/v1
export VLLM_MODEL=Qwen/Qwen2.5-14B-Instruct

# 或使用本地路径
export VLLM_MODEL_PATH=/kongweiwen/models/Qwen2.5-14B-Instruct

# Embedding 模型
export EMBEDDING_MODEL_NAME=BAAI/bge-large-zh-v1.5

# LightRAG 并发配置
export MAX_ASYNC=32
```

## 性能优化建议

1. **多GPU配置**：如果有多个GPU，使用 `start_vllm_multi_gpu.sh` 脚本
2. **并发数调整**：根据GPU数量调整 `MAX_ASYNC` 环境变量
3. **显存优化**：如果显存不足，降低 `--gpu-memory-utilization` 参数
